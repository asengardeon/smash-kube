# Contribuindo para o Smash Kube 🚀

Agradecemos o seu interesse em contribuir para o Smash Kube! Este projeto é open source e valorizamos todas as formas de contribuição, desde correções de bugs até novas funcionalidades e melhorias na documentação.

## 📋 Como começar (GitHub Flow)

O Smash Kube utiliza o **GitHub Flow** para o seu desenvolvimento. Siga estes passos:

1.  **Faça um Fork** do repositório.
2.  **Crie uma branch** a partir da `main` para a sua alteração: `git checkout -b feature/minha-nova-funcionalidade` ou `fix/nome-do-bug`.
3.  **Faça as suas alterações** e certifique-se de que os testes passam: `npm test`.
4.  **Faça o commit** das suas alterações seguindo o padrão de [Conventional Commits](https://www.conventionalcommits.org/): `git commit -m 'feat: adiciona nova funcionalidade'`.
5.  **Faça o push** para a sua branch no seu fork: `git push origin feature/minha-nova-funcionalidade`.
6.  **Abra um Pull Request** da sua branch para a `main` do repositório original.
7.  **Discussão e Revisão**: A equipe revisará o seu código e poderá sugerir alterações.
8.  **Merge**: Uma vez aprovado e com os testes passando (CI), o seu PR será mesclado à `main`.

## 🛠️ Ambiente de Desenvolvimento

O projeto utiliza **Electron**, **React**, e **Tailwind CSS**.

- `npm start`: Inicia o projeto em modo de desenvolvimento.
- `npm run watch`: Compila o frontend (React) e observa por mudanças.
- `npm test`: Executa os testes unitários utilizando Jest.

## 🧪 Testes

Sempre que adicionar uma nova funcionalidade, por favor, inclua testes unitários correspondentes. O Smash Kube utiliza Jest para testes de backend (Node.js) e frontend (React).
