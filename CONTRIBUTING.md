# Contribuindo para o Smash Kube 🚀

Agradecemos o seu interesse em contribuir para o Smash Kube! Este projeto é open source e valorizamos todas as formas de contribuição, desde correções de bugs até novas funcionalidades e melhorias na documentação.

## 📋 Como começar

1.  Faça um **Fork** do repositório.
2.  Clone o seu fork: `git clone https://github.com/seu-usuario/smash-kube.git`
3.  Crie uma branch para a sua alteração: `git checkout -b feature/minha-nova-funcionalidade`
4.  Instale as dependências: `npm install`
5.  Faça as suas alterações e certifique-se de que os testes passam: `npm test`
6.  Faça o commit das suas alterações seguindo o padrão de [Conventional Commits](https://www.conventionalcommits.org/): `git commit -m 'feat: adiciona nova funcionalidade'`
7.  Faça o push para a sua branch: `git push origin feature/minha-nova-funcionalidade`
8.  Abra um **Pull Request**.

## 🛠️ Ambiente de Desenvolvimento

O projeto utiliza **Electron**, **React**, e **Tailwind CSS**.

- `npm start`: Inicia o projeto em modo de desenvolvimento.
- `npm run watch`: Compila o frontend (React) e observa por mudanças.
- `npm test`: Executa os testes unitários utilizando Jest.

## 🧪 Testes

Sempre que adicionar uma nova funcionalidade, por favor, inclua testes unitários correspondentes. O Smash Kube utiliza Jest para testes de backend (Node.js) e frontend (React).
